# IDEA-002 — UVDL Playground

**Status:** Backlog  
**Priority:** Medium  
**Created:** 2026-07-09  
**Related:** IDEA-001  

## Problem

A static documentation page may not be enough to explore components, states, typography, fonts, and composition rules.

## Proposal

Create an interactive UVDL Playground after Core and Patterns stabilize.

Possible capabilities:

- preview Elements;
- preview Patterns;
- switch font presets;
- inspect HTML;
- copy examples;
- test states;
- test responsive layouts.

## Benefits

- Faster design and frontend iteration.
- Better validation of Core decisions.
- Useful for project onboarding.

## Risks

- Could become a separate product too early.
- Requires maintenance.
- Could distract from reference implementations.

## Dependencies

- Core Elements complete.
- First Patterns complete.
- Handbook structure defined.

## Notes

Do not build before a static Handbook or reference page exists.
