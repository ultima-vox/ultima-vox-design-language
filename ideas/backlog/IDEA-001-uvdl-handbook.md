# IDEA-001 — UVDL Handbook

**Status:** Backlog  
**Priority:** High  
**Created:** 2026-07-09  
**Related:** MANIFESTO.md, packages/css/README.md  

## Problem

UVDL needs a structured way to teach the language, not just list CSS files.

## Proposal

Create a Handbook that explains UVDL as a language:

```text
Foundation → Elements → Grammar → Patterns → Templates → Pages
```

The Handbook should include:

- purpose;
- rules;
- examples;
- anti-patterns;
- HTML usage;
- accessibility notes.

## Benefits

- Easier onboarding.
- Clearer design language philosophy.
- Better consistency across projects.
- Better bridge between design, frontend and content teams.

## Risks

- Could become outdated if created before Core and Patterns stabilize.
- Requires ongoing maintenance.

## Dependencies

- Core Elements complete.
- First Patterns complete.

## Notes

Do not implement before Core and first Patterns are stable.
