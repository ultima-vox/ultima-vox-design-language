# UVDL Ideas

**Status:** Active  
**Purpose:** Capture future ideas without interrupting current work.

## Rule

Core First. Ideas Never Lost.

During Core development, new ideas do not change the current roadmap unless they fix a critical architectural issue.

Ideas are captured here and reviewed after the current milestone is complete.

## Structure

```text
ideas/
  backlog/
    IDEA-000-example.md
  accepted/
  archive/
```

## Idea Lifecycle

- `backlog` — captured but not scheduled.
- `accepted` — approved for future work.
- `archive` — rejected, merged, or no longer relevant.

## Review Timing

Review the backlog after major milestones:

- Core Elements complete;
- first Patterns complete;
- Handbook MVP complete;
- Reference implementation complete.
